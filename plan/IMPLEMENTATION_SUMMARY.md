### FeatureRegistry.kt (预期内容)

```kotlin
// Auto-generated by KSP. Do not modify.
// Generated at: [timestamp]

package com.wgt.architecture.di.generated

import com.wgt.architecture.di.Lifecycle
import com.wgt.architecture.feature.registerFeature
import com.wgt.feature.permission.PermissionFeature

/**
 * Auto-generated feature registration function.
 * 
 * Call this function during application initialization to register all features.
 * 
 * Usage:
 * ```kotlin
 * fun initializeApplication() {
 *     InitManager()
 *     registerAllFeatures()
 *     manager.feature.initialize()
 * }
 * ```
 */
fun registerAllFeatures() {
    registerFeature<PermissionFeature>(Lifecycle.SINGLETON) {
        PermissionFeature()
    }
}
```

## 迁移其他 Feature

按照相同的步骤迁移其他 Feature：

### 1. 添加注解

```kotlin
@FeatureProvider
class MediaFeature : Feature() {
    override val name = "MediaFeature"
}
```

### 2. 删除手动注册文件

删除 `MediaFeatureInit.kt` (如果存在)

### 3. 配置模块

在对应模块的 `build.gradle.kts` 中添加 KSP 配置：

```kotlin
plugins {
    id("com.google.devtools.ksp")
}

kotlin.sourceSets.commonMain {
    kotlin.srcDir("build/generated/ksp/metadata/commonMain/kotlin")
}

dependencies {
    add("kspCommonMainMetadata", project(":ksp-processor"))
    add("kspAndroid", project(":ksp-processor"))
    add("kspIosArm64", project(":ksp-processor"))
    add("kspIosSimulatorArm64", project(":ksp-processor"))
}

tasks.withType<org.jetbrains.kotlin.gradle.dsl.KotlinCompile<*>>().all {
    if (name != "kspCommonMainKotlinMetadata") {
        dependsOn("kspCommonMainKotlinMetadata")
    }
}
```

## 故障排查

### 问题 1: 生成的代码找不到

**解决方案:**
```bash
./gradlew clean
./gradlew :feature-permission:kspCommonMainKotlinMetadata
```

### 问题 2: 编译错误 "Unresolved reference"

**解决方案:**
确保 KSP 任务在编译任务之前执行：
```kotlin
tasks.withType<org.jetbrains.kotlin.gradle.dsl.KotlinCompile<*>>().all {
    if (name != "kspCommonMainKotlinMetadata") {
        dependsOn("kspCommonMainKotlinMetadata")
    }
}
```

### 问题 3: Feature 未注册

**解决方案:**
1. 确认类上有 `@FeatureProvider` 注解
2. 确认类继承自 `Feature` 或 `IFeature`
3. 检查 KSP 日志输出
4. 验证 `registerAllFeatures()` 被调用

## 技术要点

### KSP 处理器运行在 JVM 上

KSP 处理器模块使用 `kotlin("jvm")` 是正确的，因为：
- KSP 处理器在编译时执行，必须运行在 JVM 环境
- 处理器本身生成的代码会被 KMP 项目使用
- 这是 KSP 的标准做法

### 多平台支持

KSP 生成的代码通过以下方式支持多平台：
- 生成的代码放在 `commonMain` 源集
- 通过 `kspCommonMainMetadata` 任务生成
- 各平台目标 (Android, iOS) 都能访问生成的代码

### 类型安全

- KSP 编译时验证 Feature 类型
- 生成的扩展属性使用具体类型
- 避免运行时类型转换
